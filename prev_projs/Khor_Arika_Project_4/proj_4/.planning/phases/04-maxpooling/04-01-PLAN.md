<task type="auto" tdd="true">
  <name>Task 2: Implement Fused Convolution and Max-Pooling Kernel</name>
  <files>
    - "Khor_Arika_Project_4/Problem_4/convolution_maxpooling.cu"
  </files>
  <behavior>
    - The kernel should first compute the convolution result for a tile, identical to the logic in Phase 3.
    - After the convolution result is computed and stored in shared memory, the kernel should perform a 5x5 max-pooling operation with a stride of 1.
    - The maximum value calculation must be initialized with `INT_MIN` from `<limits.h>`.
    - The final max-pooled value for each output element should be written to the output matrix in global memory.
  </behavior>
  <action>
    Modify the `convolution_kernel` in `Khor_Arika_Project_4/Problem_4/convolution_maxpooling.cu`.
    1. After the existing convolution logic and `__syncthreads()`, add the max-pooling logic.
    2. Each thread will calculate the max value in a 5x5 window centered around its corresponding element in the convolved result (which is in shared memory).
    3. Initialize a variable `max_val = INT_MIN`.
    4. Loop through the 5x5 window and update `max_val` with `max(max_val, window_element)`.
    5. Write the final `max_val` to the correct position in the output array `d_mat_out`.
    6. Ensure the host code (`main` function) correctly calls the kernel and handles memory allocation for the final output size. The output dimensions will be the same as the input dimensions because of the pre-padding and stride of 1.
  </action>
  <verify>

    </automated>
  </verify>
